# Topic 10: Inheritance and Polymorphism

## Focus
Students model commonality and variation using inheritance hierarchies and polymorphic behavior.

## Learning outcomes
Students should be able to:
- define superclass/subclass relationships appropriately;
- override methods correctly;
- use base-type references to invoke subclass behavior;
- explain when inheritance is suitable vs composition.

## Core content
- `extends`, constructor chaining (`super`)
- method overriding and dynamic binding
- polymorphic collections (intro level)
- design cautions against deep inheritance chains

## In-class activities
- shape or staff hierarchy example;
- override `toString()` and behavior methods;
- process mixed objects via common superclass/interface type.

## Common pitfalls
- forgetting `@Override`;
- calling subclass members through base reference incorrectly;
- forcing inheritance where has-a design is cleaner.

## Folder usage
Use the `src` directory for inheritance and polymorphism examples.

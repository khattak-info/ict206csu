# Topic 05: Methods

## Focus
Students decompose programs into reusable methods with parameters and return values.

## Learning outcomes
Students should be able to:
- define and call methods;
- distinguish between `void` and value-returning methods;
- pass arguments and receive return values;
- document method purpose, inputs, and outputs.

## Core concepts
- method signature
- parameter list and argument matching
- return type and `return` statement
- local variable scope
- method overloading (intro level)

## In-class activities
- extract repeated code into methods;
- create utility methods (e.g., `maxOfTwo`, `isEven`);
- simple menu program using method decomposition.

## Testing focus
- test methods independently with representative inputs;
- include edge cases (0, negatives, boundaries);
- compare expected vs actual output.

## Common pitfalls
- missing return statement in non-void method;
- parameter type mismatch;
- overusing global/static state in beginner designs.

## Folder usage
Use the `src` directory for method-focused examples.

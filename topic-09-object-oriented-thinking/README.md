# Topic 09: Object-Oriented Thinking

## Focus
Students strengthen OO design by identifying responsibilities, relationships, and abstractions before coding.

## Learning outcomes
Students should be able to:
- identify candidate classes from problem statements;
- assign responsibilities using cohesion and low coupling principles;
- represent associations in simple UML diagrams;
- evaluate and improve basic class designs.

## Core discussion
- thinking in nouns (objects) and verbs (behaviors)
- separation of concerns
- composition and has-a relationships
- design trade-offs for maintainability

## In-class activities
- transform procedural solution into OO design;
- group design critique using UML sketches;
- implement one refined class collaboration example.

## Reflection prompts
- Which class owns this behavior?
- What data should be private?
- Can this design scale with new requirements?

## Folder usage
Use the `src` directory for OO design-to-code examples.

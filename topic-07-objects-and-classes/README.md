# Topic 07: Objects and Classes

## Focus
Students move from procedural code to class-based modeling using objects, fields, constructors, and methods.

## Learning outcomes
Students should be able to:
- define a class with meaningful attributes and behavior;
- instantiate objects with constructors;
- apply encapsulation with access modifiers;
- design simple UML class diagrams and map them to Java code.

## Core content
- class vs object
- fields, constructors, instance methods
- getters/setters and validation basics
- `toString()` for object state display

## In-class activities
- model simple domain entities (Student, Book, Account);
- translate a UML sketch into a Java class;
- create driver classes to test object behavior.

## Common pitfalls
- confusing class-level and object-level responsibilities;
- exposing mutable fields directly;
- missing constructor initialization.

## Folder usage
Use the `src` directory for class and object modeling examples.
